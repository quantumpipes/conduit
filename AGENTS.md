# AGENTS.md: conduit

Internal infrastructure automation for on-premises AI deployments. Bash + Makefile toolkit for DNS (dnsmasq), TLS (Caddy + internal CA), service routing, and hardware monitoring. FastAPI admin API and a React admin dashboard sit on top of the shell commands. Apache 2.0.

## Stack

- Bash 4.0+ with `set -euo pipefail`, Makefile orchestration.
- Python 3.14+ (FastAPI `server.py`, type-hinted) wrapping the shell commands.
- React 19 + TypeScript admin UI under `ui/` (Vitest, 225 tests, 97%+ coverage).
- Caddy (internal CA, per-service TLS 1.3), dnsmasq (DNS resolver).
- Docker Compose for orchestration. Optional: `qp-capsule` for sealed audit entries.

## Commands

```bash
# Shell entry points
./conduit-setup.sh                                 # init dnsmasq, Caddy, internal CA
./conduit-register.sh --name <n> --host <ip:port> [--health <url>] [--no-tls]
./conduit-deregister.sh --name <n>
./conduit-status.sh
./conduit-monitor.sh [--server <ip>] [--containers]
./conduit-certs.sh [--rotate <n>] [--inspect <n>] [--trust]
./conduit-dns.sh [--flush] [--resolve <domain>]
./conduit-logs.sh

# Makefile
make conduit-setup | register | deregister | status | monitor | certs | dns
make dev | go | stop | logs | refresh                # Docker
make ui | ui-build | ui-typecheck                    # React UI
make test | test-unit | test-integration | test-smoke | test-ui
make check                                           # all tests + typecheck
```

## Style

- **No em dashes** in code, commits, or docs. Use commas, periods, colons, parentheses.

Bash:
- `set -euo pipefail` at the top of every script.
- `local` on every function variable. Quote all expansions.
- Validate inputs with `^[a-zA-Z0-9_-]+$`. Never `eval`. shellcheck-clean.

Python:
- Type hints on all function signatures.
- `subprocess.run(..., capture_output=True, timeout=...)`. **Never** `shell=True`.

TypeScript:
- Strict mode. Semantic color tokens from the theme (no hardcoded hex).
- Tests colocated as `*.test.tsx`, AAA pattern.

## Git workflow

- Conventional commits with scope: `feat(dns): ...`, `fix(tls): ...`, `docs(compliance): ...`.
- **Never** include `Co-Authored-By`, `Generated by`, or any AI attribution.
- `make check` before PR. Shell, UI, and API changes each need their own tests.

## Boundaries

**Always:**
- Keep the internal CA private and Caddy-managed. Auto-renew certs.
- Preserve the `services.json` schema and the audit-log conformance vectors.
- Split-tunnel assumption: services are reachable only inside the private network.
- Enforce API key auth on the FastAPI admin plane.

**Ask first:**
- Changing the `.internal` / `.local` TLD convention.
- Widening the CLI surface (new `conduit-*.sh` scripts).
- Shipping the admin UI to the public internet (design assumption is internal only).
- Adding a new cross-cutting runtime dependency.

**Never:**
- Use `eval` or `shell=True`.
- Expose dnsmasq to untrusted networks.
- Introduce deprecated crypto. Allowlist: SHA3-256, Ed25519, AES-256-GCM, TLS 1.3 via Caddy. Forbidden: MD5, SHA1, RSA (except for legacy cert interop, explicitly flagged), ECDSA-P256, 3DES, RC4.
- Skip input validation on service names, hostnames, or ports.
